# 給 Codex 的「續補篇目」prompt 範本

把下面整段貼給 Codex（或任何 AI 編碼代理），它就能照本專案規則、逐字查證地續補篇目。
完整規則與候選清單見根目錄 [AGENTS.md](../AGENTS.md)。

## 用法
1. 在專案資料夾啟動 Codex，貼上下面的 Prompt（這一輪預設補「元曲」10 首）。
2. Codex 做完一輪、把「新增了哪幾首＋每首來源網址」列給你 → 你確認無誤 → 再 push。
3. **下一批**：把 Prompt 裡「這一輪 10 首候選」換成 AGENTS.md 候選清單的其餘 10 首即可。元曲補到約 52 首後，改補其他類（樂府／漢賦／唐詩／宋詞／詩經／古文，目標各約 52）。

## 注意
- **需連網**：要抓 zh.wikisource.org / ctext.org 才能查證原文。Codex 若離線／沙箱抓不到，會（依規則第 2 條）停下來告訴你——別讓它憑記憶寫。
- Codex 是單一代理、一首一首做，慢但穩，不會像並行那樣爆用量。

---

## Prompt（複製這整段）

```text
請先讀專案根目錄的 AGENTS.md、DEVLOG.md、ROADMAP.md，理解規則與現況，再開始。

【任務】把「元曲」這一類從目前 6 首往上補。這一輪只補 10 首（小批，重品質與正確，不求快）。

【絕對規則 — 違反就等於白做】
1. 原文一律逐字查證：每一首都要實際抓取 zh.wikisource.org（維基文庫）或 ctext.org 的頁面，逐字照抄原文、並核對作者與朝代。禁止憑記憶默寫或改字。
2. 若你的環境無法連網、或抓不到可靠來源 → 立刻停止並告訴我，「絕對不要」憑印象編造原文。
3. 全程繁體中文、標點一律全形（，。：；「」〈〉《》——）。
4. 維持單檔：直接編輯 index.html 裡的 CORPUS.yuanqu.items 陣列，不要新增外部檔、不要拆成 corpus.json。

【這一輪 10 首候選】（括號內作者僅為搜尋線索，一切以查證到的來源為準；抓不到就從 AGENTS.md 的候選清單其餘項替換；勿與已收錄重複）
1. 山坡羊·驪山懷古（張養浩）
2. 天淨沙·春（白樸）
3. 沉醉東風·漁夫（白樸，「黃蘆岸白蘋渡口」）
4. 壽陽曲·遠浦帆歸（馬致遠）
5. 壽陽曲·瀟湘夜雨（馬致遠）
6. 大德歌·秋（關漢卿）
7. 水仙子·尋梅（喬吉）
8. 賣花聲·懷古（張可久，「美人自刎烏江岸」）
9. 人月圓·山中書事（張可久）
10. 折桂令·九日（張可久，「對青山強整烏紗」）

【每首要產生的欄位】（格式見 AGENTS.md；風格深度請比照 index.html 內既有條目，例如馬致遠〈天淨沙·秋思〉、喬吉〈滿庭芳·漁父詞〉）
title（曲牌·題）／ dynasty:"元" ／ author ／ text（查證後原文，用 \n 逐句換行）／ best（須為 text 中實際出現的一整句）／ notes（曲牌＋難字／典故）／ trans（白話翻譯）／ appr（賞析 3–5 句）／ bio（作者與背景 2–3 句）／ reflect（一句溫暖、扣回日常生活的提問）

【做法】
1. 逐首處理：抓來源 → 確認原文與作者 → 寫六欄。抓不到的就跳過、換候選，記下來。
2. 把查證通過的 N 首，依現有格式加進 index.html 的 yuanqu items 末尾（最後一首後補逗號、維持縮排與 JS 物件寫法）。
3. 驗證：
   (a) 語法：node -e "const h=require('fs').readFileSync('index.html','utf8');const re=/<script>([\s\S]*?)<\/script>/g;let m;while((m=re.exec(h)))new Function(m[1]);console.log('JS OK')"
   (b) 起一個靜態 http server 開 index.html，確認 console 無錯誤、元曲篇數＝6＋N、前後翻頁正常。
   (c) 隨機抽 1–2 首，把原文再對一次來源。
4. git commit（訊息註明「元曲 +N 首，逐字查證」）。先不要 push——先把「這批新增了哪幾首、每首的 source_url、有沒有候選查不到而替換」列給我看，我確認後再 push。

完成後回報：新增哪幾首、元曲總數、各自來源網址、被替換掉的候選。
```
